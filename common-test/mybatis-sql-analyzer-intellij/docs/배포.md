# 배포 가이드

> **프로젝트 루트 기준:** `common-test/`
> 모든 명령어는 프로젝트 루트(`settings.gradle` 위치)에서 실행한다.

---

## 모듈 구성 및 배포 방식

| 모듈 | 성격 | 배포 방식 |
|------|------|----------|
| `rule-core` | Java 라이브러리 (정적 분석 엔진) | `publishToMavenLocal` / Maven 저장소 |
| `rule-gradle-plugin` | Gradle 플러그인 | `publishToMavenLocal` / Maven 저장소 |
| `mybatis-sql-analyzer-core` | Java 라이브러리 (SQL 분석 엔진) | `publishToMavenLocal` / Maven 저장소 |
| `mybatis-sql-analyzer-intellij` | IntelliJ 플러그인 | `.zip` 파일 빌드 → 수동 설치 |

---

## 1. rule-core / rule-gradle-plugin 배포

### mavenLocal 배포 (로컬 테스트 용도)

```bash
./gradlew :rule-core:publishToMavenLocal
./gradlew :rule-gradle-plugin:publishToMavenLocal

# 또는 한 번에
./gradlew :rule-core:publishToMavenLocal :rule-gradle-plugin:publishToMavenLocal
```

설치 경로:
```
~/.m2/repository/com/example/rule-core/0.1.3/
~/.m2/repository/com/example/rule-gradle-plugin/0.1.3/
```

`sample-target` 등 연동 프로젝트는 `repositories { mavenLocal() }` 선언 후 사용 가능하다.

## 2. mybatis-sql-analyzer-core 배포

> **주의:** 현재 `mybatis-sql-analyzer-core/build.gradle`에 `maven-publish` 플러그인이 없다.
> 배포하려면 아래 설정을 먼저 추가해야 한다.

### build.gradle 수정

```groovy
plugins {
    id 'java-library'
    id 'maven-publish'   // ← 추가
}
```

```groovy
// 파일 하단에 추가
publishing {
    publications {
        mavenJava(MavenPublication) {
            from components.java
        }
    }
}
```

### mavenLocal 배포

```bash
./gradlew :mybatis-sql-analyzer-core:publishToMavenLocal
```

설치 경로:
```
~/.m2/repository/com/example/mybatis-sql-analyzer-core/0.1.0/
```

## 3. mybatis-sql-analyzer-intellij 배포

IntelliJ 플러그인은 Maven 배포 방식이 아니라 **ZIP 파일로 빌드**하여 배포한다.

### ZIP 빌드

```bash
./gradlew :mybatis-sql-analyzer-intellij:buildPlugin
```

**빌드 결과물 위치:**
```
mybatis-sql-analyzer-intellij/build/distributions/mybatis-sql-analyzer-intellij-0.1.0.zip
```

버전은 `mybatis-sql-analyzer-intellij/build.gradle`의 `version` 값을 따른다.

### 팀원 설치 방법 (수동 설치)

1. 위 ZIP 파일을 팀원에게 전달 (Slack, 공유 드라이브 등)
2. IntelliJ IDEA 실행
3. `Settings (Ctrl+Alt+S)` → `Plugins`
4. 우측 상단 ⚙️ 아이콘 클릭 → **`Install Plugin from Disk...`** 선택
5. 전달받은 `.zip` 파일 선택 → `OK`
6. IntelliJ 재시작

### 플러그인 업데이트 시

버전 변경 후 동일하게 `buildPlugin` 실행 → 새 ZIP 파일 배포 → 팀원이 기존 플러그인 제거 후 재설치.

> 기존 플러그인 제거: `Settings` → `Plugins` → 설치된 플러그인 목록에서 `Uninstall` 후 재설치
